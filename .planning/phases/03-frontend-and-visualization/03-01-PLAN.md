```typescript
// resolveChartRefs replaces [chart:UUID] tokens with base64 data URIs
function resolveChartRefs(markdown: string, chartMap: Record<string, ChartOutput>): string {
    return markdown.replace(/\[chart:([a-f0-9]+)\]/g, (_match, id: string) => {
        const chart = chartMap[id];
        if (!chart) return _match;
        return `![${chart.title}](data:image/png;base64,${chart.image_base64})`;
    });
}
```

   ```typescript
   export function resolveChartRefs(markdown: string, chartMap: Record<string, ChartOutput>): string {
       return markdown.replace(/\[chart:([a-f0-9]+)\]/g, (_match, id: string) => {
           const chart = chartMap[id];
           if (!chart) return _match;
           return `![${chart.title}](data:image/png;base64,${chart.image_base64})`;
       });
   }
   ```

2. **allowDataUrls function** — CRITICAL FIX for react-markdown 10.x. The `defaultUrlTransform` blocks `data:` URIs. Must pass custom `urlTransform`:
   ```typescript
   import Markdown, { defaultUrlTransform } from "react-markdown";
   import remarkGfm from "remark-gfm";

   function allowDataUrls(url: string): string {
       if (url.startsWith("data:")) return url;
       return defaultUrlTransform(url);
   }
   ```

3. **ReportView component** — Reads `state.executiveSummary` and `state.chartMap` from `useSwarm()`. Resolves chart refs via `useMemo`. Renders with `<Markdown>`:
   ```typescript
   export default function ReportView() {
       const { state } = useSwarm();
       const resolved = useMemo(
           () => state.executiveSummary ? resolveChartRefs(state.executiveSummary, state.chartMap) : "",
           [state.executiveSummary, state.chartMap]
       );

       if (!state.executiveSummary) return null;

       return (
           <div className="fade-in-up" style={{ fontSize: 14, lineHeight: 1.8, color: "var(--text-primary)" }}>
               <Markdown
                   remarkPlugins={[remarkGfm]}
                   urlTransform={allowDataUrls}
                   components={{
                       img: ({ src, alt }) => (
                           <img
                               src={src}
                               alt={alt ?? ""}
                               style={{ maxWidth: 500, display: "block", margin: "16px auto" }}
                           />
                       ),
                   }}
               >
                   {resolved}
               </Markdown>
           </div>
       );
   }
   ```

4. **Typography** — The report should feel like a professional analyst brief. Use:
   - `fontSize: 14`, `lineHeight: 1.8` on the wrapper
   - `color: "var(--text-primary)"` for readable body text
   - Let react-markdown handle heading hierarchy naturally (h1, h2, h3 from the markdown content)
   - Charts: `maxWidth: 500`, `display: block`, `margin: 16px auto` (centered, no borders, no captions)

5. **No card wrapper** — The report renders directly as page content (no `.card` class wrapping), per CONTEXT.md: "Full markdown report is the entire page content after transition."
