**ReportChat.tsx** -- Create a self-contained ChatGPT-style chat component:

1. Props: `{ reportMarkdown: string }` -- the full report text for context.
2. Local state:
   - `messages: Array<{ id: string, role: "user" | "assistant", content: string }>` -- conversation history
   - `input: string` -- current input text
   - `streaming: string` -- partial assistant response being streamed
   - `isLoading: boolean` -- whether a response is currently streaming
3. On submit (Enter key or send button click):
   - Add user message to `messages`
   - POST to `${import.meta.env.VITE_API_URL}/chat` with body:
     ```json
     {
       "message": "<user input>",
       "history": [/* previous messages mapped to Gemini format: role "user"/"model", parts: [{ text }] */],
       "report_context": "<reportMarkdown>"
     }
     ```
   - Read the response as a stream using `response.body.getReader()` + `TextDecoder`. Parse each SSE line (lines starting with `data: `) to extract `{ token, done }`. Accumulate tokens into `streaming`. When `done: true`, move `streaming` into `messages` as an assistant message and clear `streaming`.
4. Render assistant messages (including the in-progress streaming one) using `react-markdown` with `remarkGfm` plugin (same setup as ReportView.tsx -- import from same packages). Use the existing `allowDataUrls` pattern for urlTransform.
5. User messages render as plain text in accent-colored bubbles.
6. Auto-scroll to bottom on new messages using a `useRef` on a sentinel div + `scrollIntoView`.
7. Input area at bottom with: text input, send button (Send icon from lucide-react). Disable send while `isLoading`.
8. Show 2-3 suggestion chips above the input when `messages.length === 0`: "Summarize the key risks", "What sectors should I rebalance?", "Explain the contradictions". Clicking a chip sends that message.
9. If `VITE_API_URL` is not set, show a subtle "Chat unavailable -- backend not connected" message instead of the input area.
