  <files>agents/alternatives_agent.py</files>
  <read_first>agents/alternatives_agent.py</read_first>
  <action>
On lines 164-169, the current code is:

```python
crypto_prices, change_7d = await asyncio.gather(
    fetch_crypto_prices(CRYPTO_TICKERS),
    return_exceptions=False,
)
# asyncio.gather with a single coroutine that returns a tuple
prices, changes = crypto_prices
```

This is broken: `asyncio.gather` with one coroutine returns a list of one element. The subsequent unpack on line 169 is also wrong (it tries to unpack the gather result again).

Replace all 6 lines (164-169) with a single direct await:

```python
prices, change_7d = await fetch_crypto_prices(CRYPTO_TICKERS)
```

`fetch_crypto_prices` returns `tuple[dict[str, float], dict[str, float]]` so the direct unpack works correctly.

Then update the variable references downstream: the old code used `prices` and `changes` after line 169. After the fix, the two variables are `prices` and `change_7d`. Check lines 170+ to ensure all references to `crypto_prices` or `changes` are updated to `prices` and `change_7d` respectively. The rest of the function already uses these names so this should be consistent.
